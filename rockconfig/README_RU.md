# rockconfig

Загрузчик конфигурационных файлов для Go с автоматическим маппингом в snake_case, подстановкой переменных окружения и валидацией структур.

## Обзор

`rockconfig` читает конфигурационный файл (JSON или YAML), маппит его значения в типизированную Go-структуру и проверяет наличие всех обязательных полей. Теги `json` или `yaml` не нужны — имена полей автоматически преобразуются в `snake_case`.

## Быстрый старт

```go
type Config struct {
    Host string
    Port int
    Debug bool `config:",omitempty"`
}

cfg, err := rockconfig.InitFromFile[Config]("config.yaml")
if err != nil {
    log.Fatal(err)
}
```

```yaml
# config.yaml
host: localhost
port: 8080
```

## Маппинг полей

По умолчанию имена полей структуры автоматически преобразуются в `snake_case`:

| Поле в Go    | Ключ в конфиге |
|--------------|----------------|
| `Host`       | `host`         |
| `Port`       | `port`         |
| `MyField`    | `my_field`     |
| `HTTPServer` | `http_server`  |
| `UserID`     | `user_id`      |

## Тег config

Используйте тег `config` для настройки поведения поля — аналогично `bun` или `encoding/json`:

```
config:"[name][,option[,option...]]"
```

| Тег                       | Поведение                                                    |
|---------------------------|--------------------------------------------------------------|
| _(без тега)_              | snake_case имя, поле обязательное                            |
| `config:"-"`              | поле полностью игнорируется                                  |
| `config:"host"`           | явное имя `"host"`, поле обязательное                        |
| `config:",omitempty"`     | snake_case имя, поле необязательное                          |
| `config:",default=8080"`  | snake_case имя, использовать `8080` если ключ отсутствует    |
| `config:",env=PORT"`      | snake_case имя, читать значение из переменной окружения `$PORT` |

Опции можно комбинировать (см. заметку об очерёдности ниже):

```go
type Config struct {
    Host    string        // обязательное, маппинг как "host"
    Port    int           `config:",default=8080"`       // необязательное со значением по умолчанию
    Debug   bool          `config:",omitempty"`           // необязательное, без дефолта
    Secret  string        `config:",env=APP_SECRET"`      // читается из $APP_SECRET
    Timeout time.Duration `config:",default=30s"`         // поддерживает строки duration
    Token   string        `config:"-"`                   // игнорируется
}
```

> **Важно:** `default=` должен быть **последней** опцией в теге, так как после `=` потребляется весь остаток строки как значение (это позволяет использовать запятые внутри значений по умолчанию, например `default=a,b,c`).

## Валидация

После загрузки все поля без `omitempty`, `default=` и `-` проверяются на пустоту:

| Тип               | Считается пустым       |
|-------------------|------------------------|
| `string`          | `""`                   |
| `int` / `uint`    | `0`                    |
| `float`           | `0.0`                  |
| `slice` / `map`   | `nil` или длина 0      |
| `array`           | длина 0                |
| pointer           | `nil`                  |
| `bool`            | **никогда** (см. ниже) |

Если обязательные поля отсутствуют, `InitFromFile` возвращает ошибку со списком всех пропущенных полей:

```
rockconfig: missing required fields:
  Config.Host
  Config.Database.Password
```

### Поля bool

У `bool` нет осмысленного «пустого» состояния — нулевое значение `false` неотличимо от «не задано». Поэтому:

- Обычное поле `bool` **никогда** не сообщается как отсутствующее, даже если оно не указано в конфиге.
- Если нужно потребовать явной установки булева значения, используйте `*bool`:

```go
type Config struct {
    Enabled *bool // nil = не задано → ошибка валидации; &false = явный false → OK
}
```

### Pointer-поля

Pointer-типы решают проблему «нулевое значение vs не задано» для любого типа:

```go
type Config struct {
    Port    *int  // nil = не задано; &0 = явно задано как 0
    Enabled *bool // nil = не задано; &false = явно задано как false
}
```

`nil`-указатель всегда считается пустым (обязательное поле отсутствует). Ненулевой указатель никогда не пустой, независимо от значения, на которое он указывает.

## Переменные окружения

### Подстановка в файле

Ссылки `$ENV_VAR` и `${ENV_VAR}` в любом месте конфигурационного файла подставляются автоматически перед разбором:

```yaml
database:
  password: ${DB_PASSWORD}
  host: $DB_HOST
```

### Явная привязка поля

Используйте `env=VAR_NAME` для привязки поля к конкретной переменной окружения. Это имеет приоритет над значением из файла:

```go
type Config struct {
    Secret string `config:",env=APP_SECRET"`
}
```

Если `APP_SECRET` не задана в окружении, используется значение из конфигурационного файла.

## Поддерживаемые типы

| Тип Go               | Поддержка | Примечания                                       |
|----------------------|-----------|--------------------------------------------------|
| `string`             | ✅        |                                                  |
| `bool`               | ✅        | `"true"`, `"false"`, `true`, `false`             |
| `int`, `int8..64`    | ✅        |                                                  |
| `uint`, `uint8..64`  | ✅        |                                                  |
| `float32`, `float64` | ✅        |                                                  |
| `time.Duration`      | ✅        | `"5s"`, `"1m30s"` или наносекунды как int        |
| `[]T` (slice)        | ✅        | Элементы следуют тем же правилам конвертации     |
| `map[K]V`            | ✅        | Рекомендуются строковые ключи                    |
| pointer на любой тип | ✅        |                                                  |
| вложенная структура  | ✅        | Маппится как вложенный объект в конфиге          |
| `interface{}`        | ✅        | Значение передаётся как есть                     |
| встраиваемая структура | ⚠️      | Обрабатывается как именованный вложенный объект  |

## Вложенные структуры

Вложенные структуры маппятся на вложенные объекты в конфигурационном файле:

```go
type Config struct {
    Server   ServerConfig
    Database DatabaseConfig `config:",omitempty"`
}

type ServerConfig struct {
    Host string
    Port int
}

type DatabaseConfig struct {
    DSN string
}
```

```yaml
server:
  host: localhost
  port: 8080
# database опущена — разрешено из-за omitempty
```

Если `database` присутствует в файле, её поля проверяются обычным образом даже при наличии `omitempty`.

## Добавление своих форматов

Реализуйте `Decoder` и зарегистрируйте его для нужного расширения файла:

```go
type Decoder interface {
    Decode(data []byte) (map[string]interface{}, error)
}

// Зарегистрировать до вызова InitFromFile.
rockconfig.RegisterDecoder(".toml", &MyTomlDecoder{})
```

`RegisterDecoder` безопасен для конкурентного использования.

## Ограничения

- **Встраиваемые (анонимные) структуры** обрабатываются как именованные вложенные поля, а не разворачиваются. Используйте явные имена полей.
- **Нулевое значение `bool`** нельзя отличить от «не задано». Используйте `*bool` для обязательных булевых полей.
- **Нулевое значение `int`** (например, порт 0) считается отсутствующим. Используйте `*int`, если 0 — допустимое значение.
- **`default=` должен быть последним** в списке опций тега.
- **`os.ExpandEnv`** применяется к сырому файлу до разбора. Значения переменных окружения, содержащие специальные символы YAML/JSON (кавычки, фигурные скобки, переносы строк), могут нарушить структуру файла.
